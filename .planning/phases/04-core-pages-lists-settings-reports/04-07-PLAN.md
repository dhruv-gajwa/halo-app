  - `keepMounted?: boolean` (default `false` in Mantine v9 for `<Modal>` — verify against installed @mantine/core@9.2.0). When `keepMounted={false}` the Modal body is removed from the DOM on close, so React unmounts the form and RHF state resets naturally. When `keepMounted={true}` the body stays mounted; RHF retains its state across open/close cycles.

RHF v7 controlled re-default contract:
  - `useForm({ values: defaultValues })` re-applies whenever the `values` REFERENCE changes (RHF compares with `Object.is`). Recomputing `defaultValues` inline in the component body produces a NEW reference every render but RHF skips the reset when the new value is structurally equal to the previous form values (an internal optimization). The clean way to force a reset is either (a) call `form.reset(defaultValues)` explicitly in a `useEffect` that fires on the open transition, or (b) ensure the Modal contents unmount on close.
