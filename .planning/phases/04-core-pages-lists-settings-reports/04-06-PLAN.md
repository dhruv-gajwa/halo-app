<!-- dayjs ^1.11.20 (installed; verified in package.json by 04-01) -->
The UTC plugin ships in the same package. Import as `import utc from 'dayjs/plugin/utc'`. Register once via `dayjs.extend(utc)`. After registration, two call shapes are equivalent for the cases here:
  - `dayjs(value).utc().format(...)` — parse instant, then format in UTC
  - `dayjs.utc(value).format(...)` — parse as UTC from the start
For ISO strings ending in `Z` (all our stored dueDate / createdAt / completedAt values), both produce the same display. Prefer `dayjs.utc(value)` when the value is the *source* (semantic: "treat this as UTC"), and `dayjs(value).utc()` when the value is a Date object whose UTC-ness is incidental.
