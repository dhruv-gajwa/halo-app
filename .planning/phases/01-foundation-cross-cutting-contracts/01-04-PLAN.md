<objective>
Mount the full provider stack at the React tree root in the exact order `Storage → Auth → Workspace → PendoBridge → Router` (FND-07), with each provider as a thin stub that downstream phases will fill in. The Router slot is the children of `PendoBridge` — Plan 05 will place `<RouterProvider>` there. In Phase 1, the children is a temporary placeholder that the route plan replaces.

Crucially: `PendoBridge` is a **no-op pass-through stub** — `function PendoBridge({ children }) { return <>{children}</> }`. NO `pendo.initialize` call. NO `window.pendo` access. NO anonymous visitor ID generation. NO env var lookup. The slot is reserved for Phase 6 wiring; the component exists only to satisfy FND-07's provider-order requirement so the future Phase 6 wiring is a body-only change.

Purpose: FND-07 requires the provider stack to exist in the prescribed order before any feature plan starts hanging real state on it. Creating thin shells now means Phase 2's auth implementation, Phase 4's workspace settings persistence, and Phase 6's Pendo wiring are all "fill the body" tasks — none of them rewrite `App.tsx`.

Output:
- `StorageProvider` with `useStorage()` hook
- `AuthProvider` with `useAuth()` hook returning `{ user: null, signIn, signOut }` (where signIn/signOut are no-op functions in Phase 1)
- `WorkspaceProvider` with `useWorkspace()` hook returning `{ workspace: null, switchWorkspace }` (no-op)
- `PendoBridge` — pure pass-through, body is `return <>{children}</>`
- `src/App.tsx` rewritten to assemble the stack inside `<MantineProvider>` per RESEARCH.md Pattern 1
- A temporary `<Title>Halo</Title>` body (or similar) inside `PendoBridge` so the dev server still renders a visible page — Plan 05 replaces this with `<RouterProvider>`

This plan does NOT install `zustand` — Phase 2 will install it when it actually needs persistent auth/workspace state. Phase 1 uses plain React Context.

This plan does NOT add any Pendo SDK types, env var typing, anonymous ID logic, or Pendo Snippet markup. Those are Phase 6.
</objective>
