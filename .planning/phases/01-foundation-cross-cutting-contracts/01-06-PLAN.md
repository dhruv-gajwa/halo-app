<objective>
Build the centralized `PENDO_IDS` registry and four UI primitive wrappers (`Button`, `TextInput`, `PasswordInput`, `Anchor`) that forward typed `data-pendo-id` attributes to the rendered DOM, plus a `/sandbox` smoke-render route that visibly demonstrates the markup contract is working. Document the three Phase 1 markup conventions in `docs/CONVENTIONS.md`: PENDO_IDS registry as the only source of `data-pendo-id` (PEN-07), no canvas-backed charts ever (PEN-08), and password inputs carry `.pendo-sr-ignore` (PEN-09).

Crucially: this plan ships the markup CONTRACT that Phase 6 will plug Pendo into. No `pendo.initialize`, no `window.pendo` access, no `import.meta.env.VITE_PENDO_API_KEY` lookup, no anonymous ID generation. The primitives render DOM attributes only. When Phase 6 installs the Pendo snippet and calls `initialize`, the agent will discover the existing `data-pendo-id` attributes without Phase 1 needing any changes.

Purpose:
- PEN-07: The `PENDO_IDS` registry is the only source of `data-pendo-id` values; UI primitives forward without baking values in. This must be in place before Phase 2 starts building signup forms — otherwise developers reach for raw Mantine `<Button>` + hand-typed `data-pendo-id="..."` strings, and the registry discipline never establishes.
- PEN-08: Document the convention that all charts must be SVG. Phase 1 ships no charts; Phase 3 will use Recharts. The convention doc prevents Phase 3 from reaching for Chart.js or Highcharts.
- PEN-09: Ship the `PasswordInput` primitive with the `.pendo-sr-ignore` class so when Phase 2 starts capturing passwords in the signup wizard, Phase 6's Session Replay agent (once installed) automatically blocks the field.

Output:
- `src/pendo/PENDO_IDS.ts` — nested `as const` registry + `PendoId` union type
- Four primitives under `src/ui/primitives/` with a barrel export
- `src/routes/public/PrimitivesSandbox.tsx` — visible smoke-render page
- `/sandbox` route registered under PublicLayout in `src/router.tsx`
- `docs/CONVENTIONS.md` — markup conventions documented
</objective>
