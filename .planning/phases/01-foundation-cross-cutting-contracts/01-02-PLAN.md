<objective>
Install Mantine 9 and wire it as the UI baseline so every downstream plan can use Mantine components instead of unstyled HTML. Define a Halo-branded theme (`primaryColor: 'indigo'`, modern sans-serif, medium radius) and apply it via `<MantineProvider>` at the React tree root. Render at least one Mantine component (`<Title>` or `<Text>`) on the placeholder page to prove the theme is reaching the DOM.

Purpose: FND-02 requires Mantine wired with a Halo-branded theme so every page looks like a real SaaS from line one. Doing this in Wave 2 (after scaffold, before providers/router) means Plan 04's provider stack will mount `<StorageProvider>` etc. INSIDE `<MantineProvider>` (per RESEARCH.md Pattern 1), and Plan 06's UI primitives can wrap Mantine components knowing the theme is already live.

Output:
- Mantine 9 installed (`@mantine/core`, `@mantine/hooks`, `@tabler/icons-react`)
- `src/theme.ts` exporting a typed `haloTheme: MantineThemeOverride`
- `src/main.tsx` importing Mantine's CSS once
- `src/App.tsx` wrapping its body in `<MantineProvider theme={haloTheme} defaultColorScheme="light">`
- `index.html` containing Mantine's `ColorSchemeScript` (server-rendered inert in pure SPA; harmless)
- A visible Mantine-styled `<Title>` on the placeholder page proving theme delivery

This plan does NOT install `@mantine/form`, `@mantine/notifications`, `@mantine/dates`, or `@mantine/charts` — those are deferred to the phases that need them (Phase 2+ for form, Phase 3+ for notifications/charts).
</objective>
