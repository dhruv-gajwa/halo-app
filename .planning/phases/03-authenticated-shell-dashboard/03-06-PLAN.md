    - `function computeKpis(tasks: Task[], nowRef: Date, range: Range): { active: number; completedInRange: number; overdue: number; completionRate: string; avgCycleTime: string }` — implements D-17 formulas exactly:
      - `active`: `tasks.filter(t => t.status !== 'done').length` (no time filter).
      - `completedInRange`: `tasks.filter(t => t.completedAt !== null && new Date(t.completedAt) >= rangeStart && new Date(t.completedAt) <= nowRef).length`.
      - `overdue`: `tasks.filter(t => t.status !== 'done' && t.dueDate !== null && new Date(t.dueDate) < nowRef).length`.
      - `completionRate`: let `createdInRangeStillOpen = tasks.filter(t => new Date(t.createdAt) >= rangeStart && new Date(t.createdAt) <= nowRef && t.status !== 'done').length`; denominator = `completedInRange + createdInRangeStillOpen`; if denominator === 0 return `'—'`; else `${Math.round((completedInRange / denominator) * 100)}%`.
      - `avgCycleTime`: for each task with `completedAt !== null` and `new Date(completedAt) >= rangeStart`, compute `(completedAtMs - createdAtMs) / 86_400_000` (days); if no such tasks return `'—'`; else `${(sum / count).toFixed(1)}d`.
    - `function computeDayBuckets(tasks: Task[], nowRef: Date, range: Range): DayBucket[]` — produce one entry per day in the window (window-size = parseInt(range)), each `{ date: 'MMM DD' or ISO, count: tasks with completedAt within that day }`. For 7d render 7 buckets; for 30d render 30 daily buckets but X-axis can display weekly ticks (Recharts handles tick density); for 90d render 90 daily buckets — Recharts will sample. Use `date` format `YYYY-MM-DD` (sortable + Recharts displays it). The Tooltip will show the full date label by default.

      - Else → `verb='created'`, `eventAt=createdAt`.
