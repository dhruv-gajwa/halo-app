    export function Step2DetailsPage(): React.JSX.Element {
      const navigate = useNavigate()
      const draft = readWizardDraft()

      // Gate: must have completed step 1 first.
      if (!hasStep(draft, 'step1')) {

    Notes:
    - **RHF + Mantine Select / NumberInput integration**: Mantine's controlled inputs use `value` + `onChange(newValue)` (not the native `onChange(event)`). RHF's `register()` assumes native event-shape inputs, so for Select / NumberInput we use the explicit `form.watch + form.setValue` pattern. This pattern is the canonical way to bridge RHF and Mantine controlled inputs as of @mantine/core@9 + react-hook-form@7. An alternative is RHF's `<Controller>` component — either is acceptable; the watch+setValue form is shorter and types cleanly. If `setValue`+`shouldValidate: false` triggers a TS error in strict mode, switch to `<Controller render={({ field }) => <Select {...field} />} />`.
    - **Type cast on Role**: `as Step2Values['role']` casts the string to the Zod enum union. Zod validates the actual value on submit; the cast just satisfies TypeScript.
    - **NumberInput value handling**: Mantine NumberInput's `value` can be a number, `''`, or `undefined`; coerce `undefined` to `''` for the input prop. On change Mantine yields `string | number` — coerce to `number`. The Zod schema's `z.number()` requirement catches `NaN` cases at submit time.
    - The `data={ROLE_OPTIONS as unknown as string[]}` double-cast is required because TS-strict-mode `as const` makes the array readonly + literal-typed; Mantine's `data?` prop wants `(string | { value, label })[]`. The cast is safe (literal strings are still strings); it just bypasses the variance check.
    - Back uses `variant="default"` (Mantine v9 default-variant button — gray-bordered) with NO `color` prop. The `color` default is the theme primaryColor (indigo); omitting the prop keeps the button gray per UI-SPEC's "Color > Forbidden uses of indigo" rule.
    - **Do NOT** use `variant="subtle"` for Back. UI-SPEC permits either `variant="subtle"` or `variant="default"`. `"default"` reads as "secondary button"; `"subtle"` reads as "tertiary button". Both pass the locked rule; `"default"` is more conventional for a wizard Back affordance.
    - **Do NOT** render the "Already have an account? Sign in" anchor — SignupShell renders it.
    - **Do NOT** add any `pendo.*` calls. The `data-pendo-id` attributes are inert markup; Phase 6 reads them.
