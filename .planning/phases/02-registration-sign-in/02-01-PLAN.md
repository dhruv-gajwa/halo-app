<objective>
Extend the Phase 1 UI primitive surface and `PENDO_IDS` registry to cover every Phase 2 interactive control. Phase 2 introduces `Select` (role / company size / industry / plan tier / use case), `MultiSelect` (top goals), and `NumberInput` (years of experience / team size) form controls. These don't exist as wrapped primitives yet — page code MUST NOT import them from `@mantine/core` directly, so we add wrappers under `src/ui/primitives/` that match the existing pattern (typed `pendoId: PendoId` prop forwarded as `data-pendo-id`).

In the same change, extend `src/pendo/PENDO_IDS.ts` with the `signup.*` and `signin.*` namespaces locked by `02-UI-SPEC.md` — adding leaf strings to the `as const` registry automatically widens the `PendoId` union via the existing `Leaves<T>` helper, so subsequent plans can reference `PENDO_IDS.signup.step1.email` etc. with full type safety.

Purpose: a single Wave 1 plan that produces the type-system contracts and registry entries every other Phase 2 plan depends on. Splitting primitives and the registry would force an artificial dependency between two near-trivial plans — they live in the same conceptual layer (the `data-pendo-id` markup contract from Phase 1 Plan 06).

Output: 3 new primitive files, 1 modified registry, 1 updated barrel — all type-check clean. No page consumers yet; those land in Wave 3.
</objective>
