<objective>
Replace the Phase 1 `AuthProvider` stub (currently `signIn` throws, `signOut` is a no-op, `user` is null) with a real Zustand-backed authentication store that:

1. **Hydrates synchronously on app boot** by reading `halo:v1:session` via `readWithSchema(K.session(), SessionSchema, null)` and resolving the referenced visitor + workspace via `authRepo.getVisitorById` / `getWorkspaceById`. Hydration runs at module-init level (NOT in a `useEffect`) so `RequireAuth` in 02-06 sees a populated `isAuthenticated` value on the very first render — no signed-in-flash-to-sign-in regression.
2. **Exposes `useAuth()`** as the canonical hook, returning `{ user, currentVisitor, currentWorkspace, isAuthenticated, signInWithCredentials, signInFromVisitor, signOut }`. The `user` alias points at `currentVisitor` so the existing Plan 01-04 stub consumers compile. The store itself is `useAuthStore` exported from `src/auth/authStore.ts` for components that want fine-grained subscriptions.
3. **`signInWithCredentials(email, password)`** — used by `/signin` (Plan 02-10). Looks up visitor by lowercased email, `verifyPassword(input, visitor.passwordHash)`, on success writes `halo:v1:session` + updates store, returns `{ ok: true }`. On any miss returns `{ ok: false, reason: 'invalid_credentials' }` — page renders the UI-SPEC-locked "Email and password don't match. Try again." Alert.
4. **`signInFromVisitor(visitor, workspace)`** — used by wizard completion (Plan 02-09). Bypasses credential check (the caller just created the visitor + workspace via `authRepo`). Writes session + updates store atomically.
5. **`signOut()`** — clears store, removes session key, calls `clearWizardDraft()`. Resolves a `Promise<void>`. Does NOT navigate — the page that called it owns the redirect (matches Plan 01-04's existing stub contract).
6. **Installs Zustand** (`zustand@^5` — supports React 19; CLAUDE.md doesn't pin a major).

This plan replaces only the `body` of `src/auth/AuthProvider.tsx`, the `useAuth` hook signature in `src/auth/useAuth.ts`, and creates the new `src/auth/authStore.ts`. **`src/App.tsx` is NOT modified** — the provider position in the FND-07 stack is fixed per Plan 01-04's contract.

Wave 2 because: depends on 02-01 (PendoId — used by guards, but transitively reached via the AuthProvider exposing `useAuth`), 02-02 (Visitor / Workspace / Session types), 02-03 (authRepo + verifyPassword), 02-04 (clearWizardDraft for signOut).

Output: 1 new module (authStore), 2 modified modules (AuthProvider, useAuth), Zustand installed. AUTH-10 (refresh-survives-auth) and AUTH-11 (sign-out behavior) ship here.

    Hydration (the critical AUTH-10 piece):

    After `useAuthStore`'s `create(...)` call, but BEFORE export, run this once at module load:

    ```
    function hydrateAuthFromStorage(): void {
      const session = readWithSchema(K.session(), SessionSchema, null as Session | null)
      if (session === null) return
      const visitor = getVisitorById(session.visitorId)
      const workspace = getWorkspaceById(session.workspaceId)
      if (visitor === undefined || workspace === undefined) {
        // Orphan session — stored session references records that no longer exist.
        // Self-heal by clearing the bad session; user lands signed-out.
        removeKey(K.session())
        return
      }
      useAuthStore.setState({
        currentVisitor: visitor,
        currentWorkspace: workspace,
        isAuthenticated: true,
      })
    }

    // Run once at module-init level, NOT inside useEffect.
    // StrictMode double-mount affects React effects; module-init code runs once.
    hydrateAuthFromStorage()
    ```

        currentVisitor,
        currentWorkspace,
        isAuthenticated,

        signInFromVisitor,
